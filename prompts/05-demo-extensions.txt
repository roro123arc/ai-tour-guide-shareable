# GitHub Copilot booth prompts

Use this file during the booth flow. Explanations to attendees can be in Hebrew,
but the prompts below are in English so they can be pasted directly into
GitHub Copilot Chat in VS Code.

Public app:

```text
https://roro123arc.github.io/ai-tour-guide-shareable/app/
```

Repository:

```text
https://github.com/roro123arc/ai-tour-guide-shareable
```

## Step 1 - Understand the app and repo

Use this first, after attendees try the public app.

```text
I have this public web app:
https://roro123arc.github.io/ai-tour-guide-shareable/app/

And this repository:
https://github.com/roro123arc/ai-tour-guide-shareable

Please help me understand:
- what the app does from an attendee point of view
- what kind of data it uses
- how the repository is structured
- where the recommendation logic probably lives
- where I should look if I want to add a new feature

Keep it practical and explain it like I just joined the project.
```

## Step 2 - Add recommendation filters

Target preview URL after this step:

```text
https://roro123arc.github.io/ai-tour-guide-shareable/app/?step=2
```

Prompt:

```text
Add useful recommendation filters to the AI Tour Guide app.

The user should be able to filter by:
- Session type
- Topic
- Audience

Use the existing local JSON data.
The filters should update the visible recommendations for workshops and demo stations.

Keep the UI compact and friendly for a conference booth.
Do not add a backend or external API.

After implementing, explain what changed and how to test it.
```

## Step 3 - Add “Where should I go now?”

Target preview URL after this step:

```text
https://roro123arc.github.io/ai-tour-guide-shareable/app/?step=3
```

Prompt:

```text
Add a “Where should I go now?” feature to the AI Tour Guide app.

The user should choose:
- current location
- time available
- interest

The app should recommend one next best workshop or demo station.

The answer should include:
- name of the recommendation
- hall/location
- short reason
- approximate walking time

Use only local JSON data and existing venue/map logic.
Do not call Google Maps or any external API.

Keep the UI simple and useful for attendees at the booth.
After implementing, explain how to test it.
```

## Step 4 - Ask Copilot to find possible bugs

Use this as a reviewer/debugging moment. Do not say “we already found a bug”;
ask Copilot to inspect the app and identify possible issues.

```text
Act as a code reviewer for this AI Tour Guide app.

Look for possible bugs or UX issues in:
- recommendation flow
- mobile layout
- map and legend behavior
- repeated clicks on buttons
- search behavior
- data loading from JSON

List the top 3 issues you find.
For each issue:
- explain why it could happen
- point to the relevant code
- suggest a small safe fix

Do not rewrite the whole app.
```

Then choose one issue and ask Copilot to fix it:

```text
Implement the safest fix for issue number 1.
Keep the change small.
Explain how I can test it in the browser.
```

## Step 5 - Improve code quality and cache-safety

Use this as a “raw prompt” example: the request is intentionally not perfect,
and Copilot helps turn it into a clearer technical task.

```text
I want to make this app cleaner and also avoid cache problems when I change JS or CSS.
Can you help me think what should be improved?

Look at the repo and suggest:
- small code cleanup
- better helper functions if needed
- safer event handling
- cache-busting for app.js and styles.css
- anything that can make the app easier to maintain

First, rewrite my request as a clear technical task.
Then suggest a small safe implementation plan.
Do not change behavior.
```

After Copilot clarifies the task, continue with:

```text
Now implement the smallest safe improvement from your plan.
Keep the behavior unchanged.
Explain how to verify that the site still works.
```
