Handoff Orchestration 1.0

המקבילה ל-Fleet בעולם הסוכנים שלכם: Handoff orchestration (הגיע ל-1.0 ב-Build) מאפשר לבנות צוות של סוכנים שמעבירים עבודה ביניהם — אבל עם טופולוגיה מפורשת ו-guardrails: אתם מגדירים מראש מי יכול להעביר למי, במקום כאוס. והבונוס שסוגר את המעגל: סוכן שנבנה על Copilot SDK (או Claude Agent SDK) יכול להיכנס כמשתתף בתוך workflow של Foundry. אותה אזהרת מחיר: כל סוכן בצוות צורך טוקנים בנפרד.

אין מקבילה ישירה — וזה בכוונה

Merge הוא "המייל האחרון" של עולם הקוד, ולכן הוא חי רק בצד של GitHub. ב-Foundry המקבילה הרעיונית היא המסלול לפרודקשן: מ-local dev ל-Hosted Agent מנוהל בלחיצה — עם scaling אוטומטי, session persistence ו-deployment מבוקר. נקודה טובה ללקוח: זה בדיוק המקום שבו שני העולמות נפגשים — הקוד של הסוכן שלכם עובר merge ב-GitHub (עם Agent Merge), וה-deploy שלו רץ ב-Foundry.

המייל האחרון הוא הבזבזני ביותר: PR שנתקע על lint או טסט מתנדנד גורר היסטורית ריצה שלמה מחדש. Agent Merge סוגר את הלולאה אוטומטית, וזה ההבדל בין "סוכן שכותב קוד" ל"סוכן שמספק קוד". בפריוויו טכני מ-2.6.2026.
