Token Optimization Lab

מטרת העמדה היא להתנסות בדרך חשיבה של Agent economics: איך חוסכים tokens ועלות, איך שומרים על איכות, ואיך מכירים יכולות של Microsoft כמו GitHub Copilot, model routing, tool grounding, context redirection ו־eval gates.
רעיון המעבדה

מה מנסים ללמד

הסיפור הוא לא “להשתמש במודל זול תמיד”, אלא להבין איפה צריך מודל חזק, איפה אפשר כלי, ואיפה צריך gate שמגן על האיכות.
חיסכון ב־tokens
מזהים איפה ה־Agent מעביר יותר מדי context או משתמש במודל יקר לשלב פשוט.
שמירה על quality
חיסכון נחשב רק אם איכות נשארת מעל threshold. Cheap-but-bad הוא לא ניצחון.
כלים של Microsoft
מדגימים GitHub Copilot Agent Mode, model routing, tool grounding ו־eval gates.
החלטת ROI
אחרי שמוצאים את המינימום שעובד, מחליטים איפה המקסימום באמת שווה כסף.
Presenter script

פלואו הדמו לפי שלבים

לכל שלב יש הסבר בעל פה, מה מציגים, למה זה חשוב, וקישור לשם ה־prompt התואם בקובץ prompts.
1
הסבר על המעבדה
מה אומרים
“אנחנו לוקחים Agent שעובד, אבל יקר מדי, ובודקים איך לחסוך tokens בלי לפגוע באיכות.”
מה מציגים
המסך הגדול, המדדים, וקובץ challenge.py ב־VS Code.
למה
ממסגרים את הדמו כמעבדת optimization ולא כקסם של prompt אחד.
Prompt: Lab orientation
2
מי צורך הכי הרבה tokens?
מה אומרים
“לפני שמשפרים, לא מנחשים. קודם מוצאים איפה נשרפים tokens.”
מה מציגים
מבקשים מ־GHCP לסמן hotspots: lookup, ranking, context, final answer.
למה
כדי שהשיפור יהיה מבוסס מדידה ולא שינוי קוסמטי.
Prompt: Token hotspots
3
השינוי הראשון: לחסוך בלי לפגוע באיכות
מה אומרים
“הצעד הראשון הוא להעביר שלבים פשוטים לכלים או לוגיקה, ולא למודל הכי יקר.”
מה מציגים
model routing ו־tool grounding. מריצים ורואים cost/tokens יורדים.
למה
זה ה־quick win הכי בטוח: פחות premium calls, אותה איכות.
Prompt: First safe optimization
4
השינוי השני: איכות טובה יותר בלי לייקר
מה אומרים
“עכשיו מוסיפים control: context compact ו־eval gate כדי שלא נחסוך על חשבון איכות.”
מה מציגים
trace ברור, quality gate, והחלטה אם התוצאה מתקבלת.
למה
זה מדגים production thinking: מדידה, שליטה ואיכות.
Prompt: Quality without extra cost
5
כמה שווה לשלם על מודל איכותי?
מה אומרים
“אחרי שמצאנו את המינימום שעובד, מחליטים איפה המקסימום מוצדק.”
מה מציגים
השוואה בין low-cost path לבין premium-final-answer path.
למה
המסר העסקי: לא תמיד זול, לא תמיד הכי חזק — אלא ROI לפי שלב.
Prompt: Premium model decision
